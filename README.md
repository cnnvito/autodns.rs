# autodns

简体中文 | [English](README.en.md)

一个用 Rust + Tauri 做的桌面 DNS 小工具。

## 背景

这个项目来自一个很日常的公司网络使用场景：在公司网络下，需要通过公司的 DNS 获取部分私有解析记录；同时，日常访问又会依赖家里网络或公共 DNS 的解析结果。不同 DNS 对同一个域名可能返回不同结果，有些内部解析也不会发布到公网。

过去遇到这类情况，常见做法是频繁修改 `hosts` 文件，或者手动切换系统 DNS。autodns 是一次用 GPT vibe coding 做出来的桌面应用尝试：把本地 DNS、上游 DNS 顺序、查询历史和系统 DNS 接管放到一个界面里，减少来回改配置的麻烦。

## 工作方式

autodns 会在本地启动一个 DNS 服务，并按配置的上游顺序解析域名：

1. 请求先发给第一个上游 DNS。
2. 如果第一个上游成功返回解析记录，直接返回结果。
3. 如果第一个上游连接失败，或者没有该域名的解析记录，继续请求下一个上游。
4. 后续上游按顺序重复这个过程，直到拿到结果或全部失败。

这样可以把“公司 DNS 优先获取私有解析，家里/公共 DNS 继续处理普通解析”这类场景配置成固定规则，而不用频繁改 `hosts`。网络切换也可以顺带通过调整上游顺序来处理，但它不是这个项目最初的主要问题。

## 功能

- 本地 DNS 服务启动、停止和状态查看
- 多个上游 DNS 按顺序解析
- 上游健康状态、失败次数和延迟展示
- DNS 查询工具和查询历史
- 分流规则、代理、缓存等基础配置
- 可选系统 DNS 接管，默认关闭
- 中英文界面

## 快速开始

安装依赖：

```bash
make install
```

启动开发版桌面应用：

```bash
make dev
```

常用检查：

```bash
make check
make test
```

构建当前平台安装包：

```bash
make build
```

## 数据与安全

运行配置和查询历史保存在本地 SQLite 数据库中，桌面偏好保存在本地 `preferences.json`。

系统 DNS 接管默认关闭，只有在用户启用并确认后才会修改网络接口 DNS。启用前请确认目标 DNS 可用，并了解如何通过应用或系统网络设置恢复原始 DNS。
